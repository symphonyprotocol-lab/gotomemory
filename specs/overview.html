记忆共享系统

跨 ChatGPT、Claude、Gemini 的用户授权记忆控制面。

价值 目标 架构 数据模型 API 流程 安全 客户端 路线图

跨 AI 平台的 Memory Control Plane。

系统不只是保存和搜索记忆，而是决定哪些记忆可以被哪个客户端、哪个模型、以什么权限、在什么任务中使用，并让用户能追踪、撤销和审计每次暴露。

价值定位

系统最有价值的地方不是再做一个 memory.add/search，而是把记忆进入模型上下文之前的授权、审计和适配语义做成稳定控制层。

跨平台记忆护照

同一套个人、项目和团队记忆可以在 ChatGPT、Claude、Gemini、MCP 客户端和 API 消费方之间复用。

ChatGPT Claude Gemini MCP

Policy-first 记忆治理

搜索、读取、注入和导出是不同权限。拥有检索能力不等于能读取完整内容，更不等于能注入模型上下文。

read inject export

用户可见和可撤销

每次上下文构建都返回 decision_id、注入清单和省略原因，便于 UI 展示、审计追踪和用户撤销。

目标和边界

系统关注共享记忆层本身，不替代第三方平台的原生记忆，也不自动抓取所有聊天记录。

核心目标

  • 跨模型复用同一套用户记忆。
  • 解耦存储、检索、授权、同步和注入逻辑。
  • 提供统一 API 给扩展、桌面端、MCP 和后端服务。

记忆类型

支持偏好、事实、指令、关系、工作流、凭据提示和普通备注，并通过置信度、来源、新鲜度和状态管理生命周期。

preference fact workflow note

非目标

  • 不修改第三方模型平台内部记忆系统。
  • 不承诺所有平台都能官方写入原生记忆。
  • 不把记忆简单拼接成长 prompt。

所有平台适配器都必须经过 Gateway 和 Orchestrator，不能直接访问数据库、向量索引或加密对象存储。

数据模型

隐私控制落在 schema 约束里：正文和完整摘要加密，搜索只展示脱敏摘要，向量也按敏感派生数据处理。

memories
字段含义
tenant_id租户隔离边界，由 RLS 强制。
content_encrypted加密后的记忆正文。
summary_encrypted加密后的完整摘要。
summary_preview可安全展示的脱敏预览，可为空。
summary_sensitivity摘要自身敏感度，约束不低于 sensitivity。
sensitivitypublic < normal < private < secret。
embedding_policyallowed、redacted_only、disabled。
freshness当前状态、历史事实、长期事实或临时上下文。
memory_policies
字段含义
effectallow 或 deny；deny 只在胜出的 precedence 档内优先。
actioncreate、read、update、delete、inject、export。
memory_type / tag可空，按记忆类型或标签收窄（如禁注入 credential_hint）。
max_sensitivity该策略允许的最高敏感级别（上限闸）。
injection_modeauto、confirm、manual_only、never；确认需求由它派生。
precedence数值越小优先级越高，决定哪一档说了算。

向量索引与派生策略

按 embedding_model + embedding_dimension 分索引。embedding_policy 与 embedding 行的 source_kind 有固定映射：private 收敛为脱敏来源，secret 默认不向量化。

刷新键唯一约束

subject + predicate + scope 构成当前状态刷新键，部分唯一索引保证同一槽位只有一条 active 记忆。

配套表

  • audit_events：append-only + 哈希链。
  • memory_collections：记忆逻辑集合。
  • pending_confirmations：一次性确认 token。

API 设计

搜索、读取和上下文构建被明确拆开，避免搜索接口绕过读取和注入策略。

POST /v1/memories

创建记忆。Gateway 先校验租户、客户端和 scope，分类后再做敏感级别写入授权。

createclassify

POST /v1/memories/search

只返回 summary_preview、version、分数和访问决策，不返回完整 content；游标分页。

preview onlyno content

GET /v1/memories/{id}

读取完整内容必须带 purpose，通过 memory:read 策略，敏感读取审计；返回 version 与 ETag 供乐观锁。

purpose required

POST /v1/context/build

先策略过滤，再解密、脱敏和压缩，响应返回 decision_id 与 omitted 原因。

PATCH /v1/memories/{id}

使用乐观锁版本控制，支持内容、状态、刷新键和敏感级别更新。

DELETE /v1/memories/{id}

两阶段：即时去索引、停止检索注入并入清理队列；默认 30 天后物理清除内容和向量。

POST /v1/context/confirm

用一次性确认 token 兑换需确认记忆的可注入上下文，兑换时按当前策略复校，复用父级 decision_id。

confirm token

import / exports

批量导入须带 Idempotency-Key、降低可信度；导出需 memory:export，默认不含 secret 正文，写 export.created 审计。

统一错误模型

共用 { error: { code, message, decision_id } }；状态码含 403 policy_denied、409 version_conflict、429 rate_limited，不含正文。

关键流程

写入和注入都把授权放在内容处理链路前面，敏感信息不会先进入后续流水线再被拒绝。

记忆写入

提交候选记忆用户、客户端或授权数据源发起。
Gateway 粗粒度校验认证、客户端信任、租户和 scope。
分类和敏感度判定识别 type、freshness、sensitivity 和 embedding_policy。
细粒度写入授权Policy Engine 根据分类结果做最终判断。
加密保存和审计写入 Metadata DB、Encrypted Blob 和允许的向量索引。

上下文注入

Adapter 发送任务元数据包含平台、客户端、项目和 token budget。
Gateway 注入权限校验先确认调用主体能构建上下文。
检索候选记忆语义搜索和符号过滤组合召回。
解密前策略过滤不允许的记忆不会进入正文处理。
脱敏、压缩、注入按目标平台格式生成低权限上下文块。

安全和隐私

记忆内容、摘要、预览、向量和审计日志都被视为隐私边界的一部分。

级别 默认读取 默认注入 摘要和向量 典型内容
public 可由授权客户端读取 可自动注入 允许生成 preview 和 embedding 公开项目约定、公开资料
normal 按策略读取 可在授权范围内注入 允许生成安全预览 语言偏好、代码风格
private 需要明确用途和审计 默认注入前确认 只能基于脱敏摘要生成 embedding 雇主、职位、项目背景
secret 强确认和审计 默认禁止自动注入 默认不生成 embedding 密钥提示、高风险个人信息

Prompt Injection 防护

记忆正文与系统指令分离，注入时加入来源和可信等级；instruction 类型也需要用户确认。

日志边界

审计日志不得包含完整正文、完整摘要、embedding 原文或脱敏前预览，也不记录 conversation_excerpt 等请求体对话内容。

删除语义

即时去索引与停止使用，延迟物理擦除；审计仅留 ID 与决策元数据，兼顾被遗忘权与可审计。

认证机制

用户 OAuth 2.0 + PKCE，服务端租户级 API token + HMAC 签名防重放；令牌解析出 tenant_id 和权限 scope。

压缩出口受控

默认规则式裁剪不读外部 LLM，不引入新外泄出口；可选模型式压缩按 embedding 同级治理且延迟单列。

策略决策算法

确定性求值：先匹配、按 precedence 取胜出档、档内 deny 覆盖 allow、再按特异性选注入参数；默认拒绝兜底。

客户端和平台

每个 Adapter 都声明版本化能力清单，避免平台 API 变化时悄悄改变 payload 语义。

ChatGPT

API 模式优先使用 OpenAI Responses API，行为规则在 instructions，记忆作为带边界标记的 input 上下文。

Claude

Messages API 无状态，且只有顶层 system 参数、无 mid-conversation system 消息；记忆作为 user 轮上下文块或 tool result 注入，不伪装成 system 指令。

Gemini

行为规则使用 systemInstruction，用户授权记忆放入 contents 的独立上下文块。

MVP 和路线图

第一阶段不追求记忆算法 benchmark 全面领先，而是证明跨平台授权注入、敏感拒绝、决策可见和 MCP/API 可用。

普通记忆一致复用

一条偏好记忆可以在 ChatGPT、Claude、Gemini 三个 API adapter 中得到一致上下文。

敏感记忆默认受控

private 默认确认后注入，secret 默认进入 omitted，不返回正文或可注入上下文。

每次注入可解释

context/build 返回 decision_id、注入清单、省略原因和确认状态。

开放问题

这些问题决定产品默认形态、同步策略和平台内能力兼容边界。

默认本地优先，还是默认云同步？
本地优先更适合隐私敏感用户，云同步更适合跨设备和团队协作。MVP 可以先支持单用户云端或本地 PostgreSQL，再抽象同步层。
团队共享记忆是否进入第一版？
规格建议 MVP 暂不实现团队权限，但数据模型已经预留 team scope 和组织策略。
secret 是否允许强确认后手动注入？
当前策略是默认禁止自动注入，未来可以设计短期授权、二次确认和更强审计来支持手动注入。
是否兼容平台内原生记忆能力？
系统不替代原生记忆。兼容策略应作为 Adapter 能力声明，而不是核心模型假设。
